# 准家长成为家长的流程

## 一、入口：意向学员登记

- 准家长（或任何人）通过以下方式进入「意向学员登记」：
  - 从家长分享链接点「我也要报名」→ 带推荐人，进入意向学员登记；
  - 或教师/管理员从小程序内进入意向登记页填写。
- 提交后，该条记录出现在**教师端「意向名单」**（管理意向名单 / PROSPECTIVE LIST）。

---

## 二、教师端：看到意向提醒后怎么做

教师打开小程序 → 进入**意向名单**，看到新意向（姓名、年龄、家长电话、备注等）。

可选两条路径，二选一即可让准家长成为家长：

---

### 路径 A：家长先缴费，再确认入账并转正（推荐）

1. **教师联系准家长**  
   告知：请到小程序缴费，并说明「缴费后老师确认即可入学，再用同一手机号绑定家长」。

2. **发给准家长入口**  
   - 复制意向名单页的「**复制缴费说明（发给家长）**」，发给他们；或  
   - 发带 `from=parent` 的首页链接，让准家长从首页进入。

3. **准家长操作**  
   - 打开小程序首页（需能出现「意向学员缴费」「我是家长，绑定手机号」的入口）；  
   - 点「**意向学员缴费**」→ 输入**孩子姓名 + 登记时的家长手机号** → 上传缴费凭证并提交。

4. **教师确认**  
   - 教师进入「**财务 → 待确认缴费**」；  
   - 找到该条意向学员的缴费记录，点「**确认入账并转为正式学员**」；  
   - 系统会：创建正式学员、增加课时、将该学员设为「已开放家长绑定」（`parent_activated: true`）。

5. **准家长成为家长**  
   - 准家长再次打开小程序（同一微信）；  
   - 首页点「**我是家长，绑定手机号**」→ 输入**同一手机号**；  
   - 绑定成功 → 自动跳转家长首页，即可查看该学员的课时、作品、请假等。  
   - **此时准家长已成为家长。**

---

### 路径 B：教师直接转正（不经过缴费）

1. **教师联系准家长**  
   告知：已为其办理入学，请用登记的手机号到小程序里「绑定家长」。

2. **教师操作**  
   - 在**意向名单**找到该条意向；  
   - 点「**转为正式学员**」→ 输入购买课时数 → 确认办理；  
   - 系统会：创建正式学员、写入意向时的家长手机号，并**开放家长绑定**（`parent_activated: true`）。

3. **准家长成为家长**  
   - 准家长打开小程序（需从带 `from=parent` 的链接进，或从首页找到家长入口）；  
   - 点「**我是家长，绑定手机号**」→ 输入**意向登记时填写的手机号**；  
   - 绑定成功 → 跳转家长首页。  
   - **此时准家长已成为家长。**

---

## 三、流程小结

| 步骤 | 路径 A（先缴费） | 路径 B（直接转正） |
|------|------------------|---------------------|
| 1 | 意向学员登记 | 意向学员登记 |
| 2 | 教师看到意向，联系准家长 | 教师看到意向，联系准家长 |
| 3 | 教师发「缴费说明」/ 带 from=parent 的首页链接 | 教师在意向名单点「转为正式学员」并输入课时 |
| 4 | 准家长点「意向学员缴费」提交凭证 | - |
| 5 | 教师「财务→待确认缴费」点「确认入账并转为正式学员」 | - |
| 6 | 准家长点「我是家长，绑定手机号」输入手机号 | 准家长点「我是家长，绑定手机号」输入手机号 |
| 7 | 成为家长，进入家长首页 | 成为家长，进入家长首页 |

**要点：**

- 无论 A 还是 B，**最终都要准家长本人在小程序里用「我是家长，绑定手机号」输入与学员关联的手机号**，才能成为家长并看到该学员数据。  
- 家长入口（意向学员缴费、家长绑定）仅在**带 `from=parent` 的首页链接**或教师分享的家长端链接中显示；审核员扫码等不会显示这些入口。  
- 若教师希望**不通过手机号、直接用微信身份指定家长**，可在「编辑学员」中粘贴家长 OpenID 后点「设为该学员家长」（不依赖缴费或绑定页）。

---

## 四、代码与配置对应关系

- **意向名单**：`pages/enrollList`，数据来自云集合 `Prospective_students`。  
- **意向学员缴费**：`pages/findStudentToPay` → `pages/payment`，提交到 `Payment_logs`。  
- **确认入账并转为正式学员**：`paymentManage` 调用云函数 `confirmProspectivePayment`，创建 `Students` 并设 `parent_activated: true`。  
- **直接转正**：`enrollList` 的「转为正式学员」创建 `Students` 时已带 `parent_activated: true`，准家长可用同一手机号绑定。  
- **家长绑定**：`pages/bindParent`，云函数 `bindParent` 只允许「手机号匹配且该学员 `parent_activated === true`」的绑定。
