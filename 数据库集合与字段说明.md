# 云开发数据库：集合与字段说明

清空数据后，只需**创建以下集合**（无需预先建表，云开发会按首次写入自动建集合）。  
建议在「云开发控制台 → 数据库」中先**新建集合**（名称与下表一致），再按需插入初始数据。

---

## 1. configs（系统配置）

| 说明 | 操作 |
|------|------|
| 用于审核模式开关、全局配置 | 需**手动添加一条文档** |

**文档 ID 固定为：** `system_config`（在集合中“添加记录”时 _id 填 `system_config`）。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| isAuditMode | boolean | 否 | 是否开启审核模式。不写或 true = 开启；false = 关闭（老师/家长看真实数据） |

**示例：**  
`_id`: `system_config`，`isAuditMode`: `true`

---

## 2. Teachers（老师/管理员）

| 说明 | 操作 |
|------|------|
| 用于识别“老师”身份，拥有管理权限 | 老师微信 openid 需写入此集合 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| _openid | string | 是 | 微信用户 openid（云函数通过 getWXContext 写入或你手动填） |

**用法：** 在集合中“添加记录”，保留自动生成的 _id，在“_openid”字段填**老师的 openid**（可通过小程序「获取 OpenID」或云函数 getMyOpenid 拿到）。

---

## 3. Students（学员）

| 说明 | 操作 |
|------|------|
| 学员档案、余课、家长电话、是否开放家长绑定 | 老师端“录入新学员”会 add；编辑/消课会 update/increment |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 学员姓名 |
| parent_phone | string | 建议 | 家长电话（登录/绑定用），编辑页也兼容 phone |
| left_classes | number | 是 | 剩余课时 |
| parent_activated | boolean | 否 | 是否已开放家长绑定（true 后家长可用手机号在首页绑定） |
| createTime | 时间 | 否 | 创建时间（云函数 add 可带 serverDate） |
| enroll_date | string | 否 | 入学日期 |
| remark | string | 否 | 备注 |
| age | string | 否 | 年龄（意向转正时会带） |

**说明：** 云函数 add 时会自动带 `_openid`、`createTime`；update 时常用 `name`、`parent_phone`/`phone`、`left_classes`、`parent_activated`。

---

## 4. Parent_bindings（家长绑定关系）

| 说明 | 操作 |
|------|------|
| 记录“哪个微信 openid 绑定了哪个学员” | 家长在首页绑定手机号或老师用 OpenID 绑定时写入 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| _openid | string | 是 | 当前用户 openid（云函数会写） |
| parent_openid | string | 是 | 家长 openid（与 _openid 一致，用于识别家长） |
| student_id | string | 是 | 学员 _id（Students 表） |
| parent_phone | string | 建议 | 家长手机号（绑定时的手机号） |
| createTime / updateTime | 时间 | 否 | 云函数可写 serverDate |

**说明：** 云函数 init 通过 `_openid` 或 `parent_openid` 查到此表即判定为家长身份。

---

## 5. Prospective_students（意向学员）

| 说明 | 操作 |
|------|------|
| 意向登记名单，待缴费转正 | 意向登记页 add；缴费转正后 status 改为 converted |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 意向学员姓名 |
| phone | string | 是 | 联系电话 |
| age | string | 否 | 年龄 |
| status | string | 是 | `pending` 待处理 / `converted` 已转正 |
| source | string | 否 | 如「教师录入」 |
| referrer_id | string | 否 | 推荐人学员 _id（Students） |
| createTime | 时间 | 否 | serverDate |

---

## 6. Payment_logs（缴费记录）

| 说明 | 操作 |
|------|------|
| 家长/意向提交的缴费凭证，老师核销 | 缴费页 add；核销时 update status、confirm_time、student_id |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| student_name | string | 是 | 学员姓名 |
| amount_lessons | number | 是 | 购买课时数 |
| price | number | 是 | 金额 |
| proof_img | string | 是 | 凭证图片 cloud fileID |
| status | string | 是 | `pending` 待核销 / `confirmed` 已核销 |
| create_time | string | 否 | 提交时间 |
| confirm_time | string | 否 | 核销时间 |
| student_id | string | 二选一 | 正式学员 _id（已录入学员缴费） |
| prospective_id | string | 二选一 | 意向学员 _id（意向缴费） |

核销后会把 `student_id`、`status`、`confirm_time` 等更新写入。

---

## 7. Attendance_logs（考勤/消课/流水）

| 说明 | 操作 |
|------|------|
| 消课记录、续费核销流水、意向转正流水 | 老师消课 add；核销/转正时 add 一条流水 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| student_id | string | 是 | 学员 _id |
| student_name | string | 是 | 学员姓名 |
| date | string | 是 | 日期（如 toLocaleDateString()） |
| type | string | 否 | 如 `topup` 续费、`checkin` 消课等 |
| note | string | 否 | 备注（如「续费核销：￥xx / +n 课时」） |
| lessons_deducted | number | 否 | 扣减课时数（负数表示加课） |
| change_num | number | 否 | 消课数量（负数表示扣减） |
| work_img | string | 否 | 作品图 fileID |
| teacher_notes / brief / memo / note | string | 否 | 教师备注、简评等 |
| createTime | 时间 | 否 | serverDate |

---

## 8. Leave_requests（请假申请）

| 说明 | 操作 |
|------|------|
| 学员请假，老师审批 | 请假页 add；审批页 update status |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| student_id | string | 是 | 学员 _id |
| student_name | string | 是 | 学员姓名 |
| date | string | 是 | 请假日期 |
| reason | string | 是 | 原因 |
| status | number | 是 | 0 待审批 / 1 通过 / 2 驳回等 |
| create_time | string | 否 | 提交时间 |

---

## 最小可运行配置（清空后快速恢复）

1. **configs**：新建集合 → 添加一条，_id 填 `system_config`，字段 `isAuditMode` 填 `true`（或不填）。
2. **Teachers**：新建集合 → 添加一条，在“_openid”字段填**你的微信 openid**（用小程序或 getMyOpenid 获取），这样你就是管理员。
3. 其他集合 **Students、Parent_bindings、Prospective_students、Payment_logs、Attendance_logs、Leave_requests** 可先**只建空集合**，无需预先插数据；老师/家长正常使用“录入学员”“意向登记”“缴费”“消课”时会自动写入。

**权限建议：**  
- 若希望仅云函数可读写：在控制台将各集合权限设为“仅管理端可读写”。  
- 若希望“用户只能读写自己的数据”：需与云函数里用的 `_openid` / `parent_openid` 等字段一致，并注意 Parent_bindings 的查询方式（云函数已按 _openid、parent_openid 查）。
